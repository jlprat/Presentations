Browser Games

Only for Specialists

Browser Games... Traditionally, the option of making browser games was reserved only for specialist developers. How many of you are developers? And Front End developers? And Game Developers? I would like to demystify this belief.

Common Approaches

What comes to your mind, when you think about web games? Flash, Canvas, WebGL and the one reserved traditionally for Front End Developers... Until now...

Specialization needed

Why people think that games are only for specialists? Maybe, because of those examples we've just seen

Flash

  • New Language
  • Different Best Practices
  • "Desktop Only"
The bast majority of games to date are Flash. What's the problem with it? it's a different language (OK, descendant of ECMAScript, but different language). Lately, it encountered a big problem, mobile devices don't support them

Canvas

  • Outside of the DOM
  • Not so clear Separation Of Concerns
  • Redundancy of Styles
This is the most common HTML5 approach of building games that must run everywhere. Meanwhile it's in JS, it's not possible to take advantage of our familiar DOM, this will imply that we need to recode some of our styles again, and most importantly, write them in JS.

WebGL

  • Low Level API
  • New Terrain
  • Shaders
This is a more uncommon way. Using WebGL we unleash all the potential of our games. But, great power comes with a great responsibility. It's completely new domain for most Front End developers. And it has a problem, some devices don't support WebGL, and you need to manually provide an alternative

Game Engines that can help you

But of course, there are some game engines that can help you building games.

Starling

  • Flash Library
  • App conversion
  • Tweens, Blends, Multitouch...
www.gamua.com/starling/
One of the most common game engines / frameworks is Starling. It is a flash library, it offers some app conversion features, so your game can run in mobile devices as native app. This can speed up your. But as we agreed, this is even in another language...

Pixi.js

  • WebGL with Canvas Fallback
  • Physics Library
  • OpenSource
www.pixijs.com
Pixi.js. This game engine offers the best of canvas and WebGL, when available the game will be run in WebGL otherwise it will fallback to Canvas. It contains a physics library and it's opensource

PlayCanvas

  • WebGL only
  • Online Editor and Hosting
  • Physics Library
  • OpenSource
www.playcanvas.com
Playcanvas. If you can afford targeting only WebGL devices, this can be your friend, it has an online editor and hosting. It's also opensource and it contains a physics library as well.

Unreal

  • Powerful Engine
  • C++
  • Cross Platform
  • Physics, AI, Animation Effects...
www.unrealengine.com
Unreal. You can get all the power of Unreal Tournaments for your game. Drawback, you need to code it in C++. But you get a game for PC and gaming consoles, not bad! It has Physics, AI, animation effects libraries and so...

Tips to Improve Productivity

Of course there some tips to improve your productivity as game developers

WebSockets

  • Enables Full Duplex Communication
  • Perfect for Reactive Apps
  • Supported by All Major Browsers
Websockets, are must have, there should be an automatic assimilation between multiplayer and websockets. They are perfect for reactive apps and have full duplex communication.

Box2D

  • Physics Library
  • Models a Gravity World
  • Collision Detection
If you are not so fond in physics, this nice library can help you. It simulates a gravity world and you you can get how your objects behave in such a world in a very realistic way.

Work with your own coordinate system

  • Ideal for Responsive Games
  • Abstraction from the Real Screen Size
  • Keep Code Simple
For simplicity, I was working with percentages in the previous examples, but if you want to develop a game, you should think about getting away of the pixel world and create your own coordinates system (for example 0 to 100) this way you can get abstracted from the real screen size. However, you need to calculate some of these values in your JS and then apply inline styles. These are only some easy tips, but there are much more hints and tips you can use. Also the examples are quite easy, but our HTML5 games are done purely with DOM. I recommend you to try them out.
